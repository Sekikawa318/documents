  • kaggleのコンテストページ

データ分析の流れ¶

データ分析にはある程度決まった流れがあります．
必ずしもこれ沿う必要はありませんが，私はこの流れに沿って行っています．
本日はこの流れに沿って，４までを実際に行ってみる予定です．

1. データの把握¶

 - データの読み込み
 - データの行数・列数・csvファイル数の確認
 - データの型・欠損値の確認・全て同じ値をとる列はないか・独自のデータがあるかなど

2. ベースラインの作成¶

 - 簡単な前処理のみをしてベースとなる機械学習モデルを作る
 - 初めから全てやると分からなくなるので，雑に作った簡単な機械学習モデルをベースラインとして改良していく

3. EDA(探索的データ解析)¶

 - ベースラインの精度を改良するために，データを観察して様々な処理をかける
 - データ分析コンペの勝敗はここで決まると言って良い
 - 機械学習の知識を元にデータを処理することも重要だけど，個人的には取り組み続ける持久力と柔軟な発想力の方が重要だと思う

4. 精度確認¶

 - EDAで処理したデータでの精度を確認する
 - 精度が上がってれば意味があったと判断
 - 精度が変わらなければ意味なかったと判断
 - EDAと精度確認を繰り返して精度を上げていく(3と4の繰り返しがデータ分析の8割を占める)

5. モデルのチューニングとアンサンブル¶

 - 機械学習モデルにはそれぞれ自分で設定するパラメータがあるので，最適な値を探す(チューニング)
 - 複数のモデルを作ったなら組み合わせると精度が上がることが多い(アンサンブル)

def read_s3_csv(SRC_OBJECT_KEY_NAME, SRC_FILE_ENCODING="utf-8", SRC_BUCKET_NAME="dataanalytics-training-2021"):
    """
    S3のcsvファイルを読み込んで，pandasのdf型を返す

    Parameters
    ----------
    SRC_BUCKET_NAME : str
        S3のバケット名を指定する．
        今回は「dataanalytics-training-2021」で固定．
        その他のバケットはアクセスできないはず．
    SRC_OBJECT_KEY_NAME : str
        取得するcsvファイルのオブジェクト名を指定する．
    SRC_FILE_ENCODING：str
        csvファイルの文字エンコードの種類を指定する．
        
    Returns
    -------
   df : pandas.DataFrame
        csvファイルをデータフレームで読み込んだもの
    """

    # ファイルをロード
    s3 = boto3.resource('s3')
    src_obj = s3.Object(SRC_BUCKET_NAME, SRC_OBJECT_KEY_NAME)
    body_in = src_obj.get()['Body'].read().decode(SRC_FILE_ENCODING)

    # バッファを作成
    buffer_in = io.StringIO(body_in)

    # データフレーム化
    df = pd.read_csv(buffer_in)
    
    return df

def display_isna(df, df_name):
    """
    データフレームの欠損値を確認する

    Parameters
    ----------
    df : pandas.DataFrame
        欠損値を表示するデータフレーム
    df_name :pandas.DataFrame
        インデックスに表示するデータフレーム名
        
    Returns
    -------
   df : pandas.DataFrame
        欠損値情報を持つデータフレーム
    """
    
    temp =  pd.DataFrame(df.isnull().sum()).T
    temp.index = [df_name + "の欠損値数"]
    return temp

2.2.1. 欠損値の対処¶

欠損値を含むデータはそのままではモデル作成ができないため，何らかの対処をする必要があります．
欠損値への対処に関しては，様々な方法がありますが，今回は最も簡単な欠損値を含むデータを削除する方法を取ろうと思います．
(本来は，データを観察して扱うデータに最も適した手法を選ぶ必要があります)

参考：欠損値への対処法の例¶

  • 欠損値を含むデータを削除する
  • 欠損値を平均値などの代表値で埋める
  • 欠損値に絶対に取り得ない値を入れる(欠損値という情報を持たせる)
  • 他の特徴量を元に機械学習モデルを作成し，欠損値を予測して埋める

ラベルエンコーディング¶

例) X,Y,Zというカテゴリデータを持つ場合

  • X → 0
  • Y → 1
  • X → 2

というように同じカテゴリデータが同じ値を取るように数値に変換しています
これによって，カテゴリデータではなく，数値データとしてX, Y, Zという3種類の違いを情報として機械学習モデルに与えることができます

参考: https://vector-ium.com/label-encoding-coding/

Random Forest¶

概要¶

  • 木モデルを使った機械学習モデル
  • Decision Treeと呼ばれる単体では精度が良くないモデルを，複数集めることで単体のDecision Treeより精度の良いモデルを作る
  • 複数のモデル毎に，学習に扱うデータと扱う特徴量を変えることでモデルのバリエーションを増やしている

特徴¶

  • データの前処理が楽(スケーリングなどが不要) -> 今回このモデルを選んだ理由
  • モデルのパラメータ調整をしなくても精度が高く出やすい(初期値で十分精度が高い)
  • モデルの内部のロジックが直感的で分かりやすい


まずtrian_dfとtest_dfを説明変数と目的変数に分けましょう．

  • 説明変数：モデルの学習に使用する変数,今回の問題ではtarget以外の変数
  • 目的変数：予測対象の変数,今回の問題ではtarget

次に，trainデータを学習用データと検証用データに分けましょう．
学習に使ったデータで正解率を測ろうとすると，未知のデータに対する本当の精度が測れません．
(学習用に使ったデータを見てモデルを作成しているため，学習用データはモデルにとって未知のデータではない)

(ひとまずそういうものなんだなという理解でも良いです)

3. EDA(探索的データ解析)¶

このセクションでは，データの加工(変更・追加・削除)することで，ベースラインのスコアを上げることが目的です．
決まった手順などがあるわけではなく，データの探索をして仮説を立てながら，実際にデータを加工してスコアが上がるか試していきます．

ここでは，少ししか取りあげませんが，皆さんもいろいろ考えて取り組んでみてください.

次に，相関行列のヒートマップを表示して相関が強い特徴量の組みがないか確認しましょう．
相関行列は各成分に対応する相関係数を並べた行列のことであり，値の大小に応じて色をつけたものをヒートマップと呼びます．
この図を表示することによって，変数間の相関の強さを一目で把握することができます．

機械学習のモデルによっては,同じようなデータ(相関が強いデータ)が混ざっていると多重共線性という問題が生じる場合があります．
そのため著しく相関の強い特徴量の組みがある場合は,その特徴量のうち一つだけを残して他の変数を削除するといった対策をすることがあります．

この図から、SibSpとParchの値に比較的強い正の相関があることがわかります．

  • SibSp：同乗していた兄弟姉妹・配偶者の数
  • Parch：同乗していた親・子供の数

なので，相関があることは直感的に理解できます．
ここでSibSpの値とParchの値の和をとって「同乗していた家族の人数」という新しい変数を加えるアイデアが得られます．
SibSpとParchという不自然な分類で二つの変数に分割してあるよりも「同乗していた家族の人数」という変数の方が自然な可能性があるためです．

他にもPclassとFareの値に比較的強い負の相関が見られます．
この相関から，Pclassの値が小さいほどチケットのグレードが高いことが予想できます．

SibSpの値とParchの値の和をとって「同乗していた家族の人数」という新しい変数を加えるというアイデアで，精度向上するか試してみましょう！
